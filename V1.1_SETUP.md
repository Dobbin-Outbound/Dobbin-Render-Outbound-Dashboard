# V1.1 Setup — Real Message Analysis (Supabase + n8n + Gemini)

V1 (HeyReach-only) is live. V1.1 lights up:
- **Meetings Booked** KPI (real, from message content)
- **Positive Replies** KPI (real, from AI sentiment classification)
- **Reply Classification donut** (Interested / Open / Closed / Awaiting AI)
- **Open Conversations** count from Supabase

Everything below is **one-time setup**. Once it's done, replies flow in automatically.

## Step 1 — Deploy the schema to Supabase (5 min)

1. Open your Supabase project: https://supabase.com/dashboard/project/jezvwasrtgjrunagabpj
2. Go to **SQL Editor** → **New query**
3. Paste the contents of [`supabase_schema.sql`](./supabase_schema.sql) and run
4. Verify in **Table Editor**: you should see `heyreach_messages` table

## Step 3 — Set up the n8n workflow (~10 min)

The workflow uses **only built-in n8n nodes** (Webhook, Code, HTTP Request) — no community packages required. So the "Authorization failed" error from the HeyReach community node doesn't apply here.

The workflow has 4 nodes:

```
[Webhook]  →  [Build Supabase Payload (Code)]  →  [Upsert to Supabase (HTTP)]  →  [Respond OK]
```

**Steps:**

1. In n8n: click **+ Add Workflow** → click the menu (⋯) → **Import from File** → pick `n8n_workflow.json`
2. Click the **Webhook** node → click **Listen for Test Event** to grab the URL (or just **Save** the workflow and use the production URL — both will be shown)
3. Click the **Upsert to Supabase** node → in the Headers section, replace **both** occurrences of `PASTE_SUPABASE_SERVICE_ROLE_KEY_HERE` with the actual service role key (the same 219-char `eyJhbGc...` you set on Render)
4. Click **Save** then toggle the workflow **Active** (top-right switch)
5. **Register the webhook in HeyReach**:
   - Log into HeyReach → Settings → Integrations → Webhooks (or wherever your team set it up)
   - Add a new webhook for **"Message Received"** event (this is the reply event)
   - Paste the n8n webhook URL from step 2
   - You can also add the same URL for **"Connection Accepted"** if you want acceptance events stored too (the schema handles them)

**To test it** before waiting for a real reply:
- In n8n, click the Webhook node → **Listen for Test Event**
- In a separate terminal, send a fake payload:
  ```
  curl -X POST <your-n8n-webhook-url> \
    -H "Content-Type: application/json" \
    -d '{"conversation_id":"test123","event_type":"MESSAGE_RECEIVED","timestamp":"2026-05-11T10:00:00Z","sender":{"id":135433,"full_name":"Alaina Vincent"},"lead":{"id":"lead-1","full_name":"Test Prospect","company_name":"TestCo"},"campaign":{"id":1,"name":"Test"},"recent_messages":[{"author":"Test Prospect","body":"Sounds great, let'\''s book a call"}]}'
  ```
- You should see all 4 nodes turn green
- Open Supabase → Table Editor → `heyreach_messages` → confirm the test row exists
- Delete the test row, then activate for production

From this point on, every new HeyReach reply pushes a row into Supabase.

## Step 5 — Trigger AI evaluation on existing replies (when Gemini key arrives)

Once `OPENROUTER_API_KEY` is set on Render, run a one-time backfill:

```bash
curl -X POST https://outbound-dashboard-wai9.onrender.com/api/messages/evaluate \
  -H "Content-Type: application/json" \
  -d '{"limit": 50}' \
  --cookie "session=<YOUR_LOGIN_COOKIE>"
```

That'll classify the first 50 unevaluated conversations. Repeat until all are done. (Future replies are evaluated automatically on a schedule once we add the cron — for V1.1 launch, manual is fine.)

## What's in each file

- **`supabase_schema.sql`** — The single `heyreach_messages` table + upsert function
- **`n8n_workflow.json`** — 4-node workflow: Webhook → Get Conversation → Build Payload → Upsert
- **`supabase_client.py`** — Lightweight HTTP wrapper around Supabase PostgREST
- **`gemini_evaluator.py`** — Calls Gemini 2.5 Flash to classify conversations
- **`app.py`** — New endpoints: `/api/messages/status`, `/api/messages/stats`, `/api/messages`, `/api/messages/evaluate`

## Troubleshooting

**Validation says "Supabase schema: heyreach_messages table missing"**
→ Run `supabase_schema.sql` in the SQL editor (Step 1).

**Validation says "0 conversations stored"**
→ The schema is deployed but n8n hasn't pushed any data yet. Confirm the workflow is **active** and the HeyReach webhook is registered. Trigger a test reply manually.

**Validation says "AI sentiment classification: OPENROUTER_API_KEY not set"**
→ Expected until the client provides the key. Total Replies + Open Conversations still show real numbers; only the Interested / Booked classification stays as "Awaiting AI".

**`/api/messages/evaluate` returns errors**
→ Check the Render logs. Most common: rate limit hit (Gemini free tier is generous but not infinite). Re-run with smaller `limit`.
